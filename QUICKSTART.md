# 🚀 AI Finance Assistant - Полный проект

Дипломный проект: AI-помощник для управления личными финансами.

## 📦 Состав проекта

```
finance_assistant/   # Django Backend
finance_frontend/    # React Frontend
```

## 🛠 Быстрый старт

### 1. Backend (Django)

```bash
cd finance_assistant

# Создаём виртуальное окружение
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или venv\Scripts\activate  # Windows

# Устанавливаем зависимости
pip install -r requirements.txt

# Создаём .env файл
cp .env.example .env
# Редактируем .env и добавляем GROQ_API_KEY

# Применяем миграции
python manage.py migrate

# Создаём категории
python manage.py create_default_categories

# Запускаем сервер
python manage.py runserver
```

Backend будет доступен: http://localhost:8000

### 2. Frontend (React)

```bash
cd finance_frontend

# Устанавливаем зависимости
npm install

# Создаём .env файл (опционально)
cp .env.example .env

# Запускаем dev сервер
npm run dev
```

Frontend будет доступен: http://localhost:3000

## 💳 Новая функция: Виртуальный счёт (v3.1)

### Что добавлено
- **Имитация банковского счёта** — баланс, номер счёта, история операций
- **Автопополнение зарплатой** — в конце месяца (28/29/30/31) или в выбранный день
- **Ручное пополнение** — в любой момент
- **Запланированные платежи**:
  - Подписки — списываются автоматически в свою дату
  - Аренда/ком.услуги — оплачиваются вручную кнопкой
- **Резервирование** — запланированные платежи уменьшают «доступно», но не баланс, пока не наступит дата
- **Оплата с автокатегорией** — по ключевым словам; если не определилось, просит выбрать
- **Лимиты категорий** — предупреждение при превышении (оплата проходит)
- **Графики** — расходы по категориям, прогресс по лимитам, приход/расход за месяц
- **Кнопка «промотать на след. месяц»** — для теста начислений (т.к. реального планировщика нет)

### Новые страницы
- `/account` — Мой счёт (баланс, пополнение, оплата, настройки зарплаты, графики)
- `/scheduled` — Запланированные платежи (подписки и регулярные)

### Новое приложение backend: `apps/accounts`
Эндпоинты под `/api/accounts/`:
- `GET/PATCH account/` — счёт и настройки зарплаты (GET синхронизирует даты)
- `GET transactions/` — история
- `POST topup/` — пополнение
- `POST pay/` — оплата (с автокатегорией)
- `GET analytics/` — данные для графиков
- `POST fast-forward/` — промотать время
- `CRUD scheduled/` + `POST scheduled/<id>/pay/` — платежи
- `CRUD rules/` — правила автокатегорий

### После обновления нужно
```bash
cd finance_assistant
python manage.py migrate
python manage.py create_default_categories  # если БД новая
```
Счёт создаётся автоматически при первом открытии страницы «Мой счёт».

## 🏦 Онлайн-банк + поток доходов/расходов (v3.2)

### Архитектура
Два интерфейса над одним виртуальным счётом:
- **Онлайн-банк** (`/bank`, бренд «Amanat») — отдельный светлый дизайн, без сайдбара Finance AI. Здесь происходят деньги: пополнение (с типом: зарплата / фриланс / другой доход / перевод-не-доход), оплата, **настройки зарплаты** (автозачисление), кнопка «промотать на след. месяц».
- **Finance AI → «Мой счёт»** (`/account`) — только информация и графики + ссылка «Открыть онлайн-банк».

### Поток денег
- Пополнение в банке с типом дохода → запись в **Доходах** Finance AI (зарплата/фриланс/другое)
- Пополнение типа «Перевод (не доход)» → только баланс, в Доходах НЕ отражается
- Любая оплата со счёта → запись в **Расходах**
- **Лимиты** настраиваются только в Finance AI (на странице Бюджет)

### Промотка времени
Один клик «След. месяц» = одна зарплата (промотка до ближайшей даты зарплаты) + подписки за период. Раньше был баг с двойной зарплатой — исправлено.
